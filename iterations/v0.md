# v0 — 立项背景与待研究问题

## 背景

面试题要求：设计并实现客服对话结构化信息提取工具。

## 输入数据

- 25 条客服对话（JSON）
- 1 条提取示例（仅供参考，非标准答案）
- 复杂场景：多诉求、转人工、情绪爆发、信息缺失、话题切换、跨会话引用等

## 任务要求

1. 设计提取 Schema（字段+理由）
2. 实现提取工具（LLM API 或 mock）
3. 处理边界情况（README 说明策略）
4. 验证准确性（人工抽检 ≥5 条，报告准确率）
5. README：设计思路、任务拆解、AI 工具使用

## 研究结论（2026-06-15）

已输出研究报告，确认方案：
- Schema：分层设计（基础 + issues[] + 情绪 + 风险 + 标签）
- 技术：JSON Schema + 后校验
- 边界：多诉求拆分、情绪变化、流失风险、标签系统
